What is Networks?

A CGS-based Game:


  • This game is based on J. Brajard and T. Hilaire Coding Game Server (CGS) framework. This Github project contains a bunch of files implementing a socket-based web server that can be used to develop a Coding Game.
  • The idea behind CGS was to propose a fun end-of-semester project to undergraduate students. Being involved in game programming made it easier to learn and code concepts from Game Theory such as min-max, A*... For now, the already created games are mainly turn-based, two players games.
  • If you want more info on the project, check out the Github link.

A Web of Nodes,
a Net of Links:


  • In Networks, you are a hacker whose goal is to infiltrate a server and capture its core. To do so, you have to go through a series of computer nodes, the HackCells. Each HackCell is linked to others: it is only by hacking a node that you will get access to the adjacent nodes linked to it.
  • At the center of the web, there is the core HackCell. Just capture it to win the game. Seems fairly easy, yes?
  • But be careful! If your opponent is a good enough hacker, he may be able to destroy or create new links in the system and therefore turn the game around. Plan your path carefully, try to block your adversary as much as possible and always watch out for the unexpected...

3 Types of Nodes,
2 Hackers, 1 Winner!


  • When a grid of HackCells is created, it can contain several types of nodes that each take a certain amount of time to be conquered: diamonds are simple nodes that can be captured in 1 turn, squares need 2 turns and circles need 5 turns! Sometimes the obvious way is not the quickest...
  • Like all CGS-based games, Networks involves 2 players that compete to reach the core node. As soon as either one captured it, the game stops and the hacker that got it wins.
  • Be sure not to make any wrong move or the system will be alerted of presence and push you out, thus the other player will win.
